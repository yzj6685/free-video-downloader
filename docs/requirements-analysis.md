# 需求分析文档

## 背景

很多同学有从多个平台下载视频并保存到本地的需求，但平台自带下载能力通常存在限制，例如入口不明显、不支持批量、清晰度受限、无法在手机上方便保存，或者下载流程太繁琐。因此希望开发一个“万能下载视频”的网站，让用户可以快速、随时随地、跨平台保存视频。

## 目标用户

- 学生和自学者：保存课程、公开学习素材、讲座片段，方便离线复习。
- 内容创作者：下载本人发布或有授权的视频素材，用于剪辑、归档、复盘。
- 运营和资料整理者：保存公开素材、活动视频、案例内容，便于整理。
- 移动端用户：希望在手机浏览器里完成粘贴链接、解析和下载。

## 核心痛点

- 平台下载入口分散，不同平台规则不同。
- 很多平台不支持批量或跨设备保存。
- 用户不知道该用什么工具，工具安装门槛高。
- 手机端下载体验通常更差。
- 高级需求逐渐增多，例如批量下载、字幕提取、视频总结、字幕翻译、音频提取。

## 业务目标

- 首版用一个简单、直接、可演示的工具站验证核心需求。
- 页面要具备消费引导能力，突出免费试用和会员权益。
- 首版不做真实付费，但要让用户能感知高级能力价值。
- 文档沉淀需求和方案，方便后续 AI 或开发者继续扩展。

## 已确认决策

- 首版范围：单链接极简版。
- 技术栈：Vue 3 + Vite + Tailwind CSS + FastAPI。
- 下载能力：封装 yt-dlp。
- 部署阶段：先做本地演示版。
- 数据存储：首版不使用数据库。
- 下载交付：直链优先，失败或不稳定时后端中转。
- 商业模式：免费试用 + 会员引导。
- 扩展能力：首版只预留架构，不真正实现视频总结、字幕翻译、真实支付。
- AI 能力方向：后续优先接第三方 AI 接口。
- 真实付费闭环：第二阶段开发。

## 首版范围

首版必须实现：

- 用户粘贴单个视频链接。
- 用户粘贴包含链接的分享文案时，系统自动提取第一个 URL。
- 后端解析视频信息。
- 前端展示标题、封面、作者、时长、平台、格式选项。
- 用户选择格式并下载。
- 页面展示会员权益和高级能力入口。
- 视频总结、字幕翻译接口作为占位返回“暂未开放”。
- 会员套餐接口返回静态套餐信息。
- 沉淀 README、需求分析、方案设计文档。

## 非目标范围

首版不实现：

- 用户注册登录。
- 真实支付、订单、会员权限校验。
- 数据库存储。
- 批量下载。
- 下载历史。
- 真实视频总结和字幕翻译。
- 后台任务队列。
- 线上部署和文件缓存清理。
- DRM 绕过、破解、盗取登录态、规避会员墙或下载私密内容。

## 产品表达

可以强调：

- 万能视频下载器。
- 复制链接，一键保存到本地。
- 支持大量公开可访问的视频站点。
- 手机浏览器也能用。
- 会员将支持批量下载、高清下载、音频提取、视频总结、字幕翻译。

需要避免：

- 绕过限制。
- 破解会员内容。
- 下载私密视频。
- 盗取登录态。
- 保证所有平台 100% 下载。
- 自动读取用户浏览器 Cookie。

## 平台限制说明

抖音、B站等平台可能要求新鲜游客 Cookie 或用户主动提供的 Cookie 才能解析。下载解析链路必须明确提示这一点，并支持通过 `YTDLP_COOKIE_FILE` 配置用户主动导出的 cookies.txt。AI 字幕分析链路在本机运行时可自动尝试读取 Chrome/Edge 的 B 站登录态 Cookie 作为字幕接口兜底，但不得注入公共账号 Cookie、不得绕过会员墙或 DRM。

## 后续扩展方向

- 批量下载：粘贴多行链接、任务队列、失败重试、批量打包。
- 视频总结：优先基于字幕生成摘要；无字幕时接音频转写。
- 字幕翻译：提取字幕、翻译字幕、下载双语字幕。
- 付费闭环：登录、套餐、订单、支付回调、会员权限。
- 下载历史：记录解析与下载任务。
- 线上部署：Docker、队列、对象存储、临时文件清理、监控告警。

## 验收标准

- 用户能在首页完成单链接解析和下载。
- 页面在桌面和手机宽度下都能正常使用。
- 会员入口和高级能力入口清晰可见。
- 错误提示可理解，不暴露底层异常。
- 文案具有吸引力，但不出现高风险表述。
- 文档能让后续 AI 快速理解项目背景、边界和扩展方向。
## 当前需求补充记录：抖音无需用户 Cookie

用户明确要求抖音下载不能依赖用户手动提供 Cookie，也不能使用公共共享账号。当前实现遵循这个边界：

- 不自动读取用户浏览器 Cookie。
- 不弹窗要求用户扫码登录抖音。
- 不内置公共账号或公共 Cookie。
- 首版以服务端解析源方式支持抖音公开视频：本地演示默认尝试公开解析源，生产环境建议接自建签名解析服务或商业 API。
- 抖音图集或非视频内容不纳入首版核心下载范围，会给出明确提示。

这个方案适合网站形态：用户只需要粘贴公开视频链接，不需要暴露自己的平台登录态。浏览器插件方案仍可作为未来“本地增强版”方向，但不作为当前网站 MVP 的主路径。

## 当前需求补充记录：AI 视频分析一期

用户确认希望把“万能视频下载器”扩展为学习效率工具：解析视频后，快速了解视频大纲、核心知识点，并围绕视频内容追问。

一期范围：

- 提取平台已有字幕或自动字幕。
- 使用 DeepSeek 生成中文学习笔记，包括摘要、大纲、核心知识点和推荐追问。
- 总结摘要支持 SSE 流式输出，先让用户看到内容，再补齐完整学习笔记。
- 支持基于当前视频分析结果进行 AI 问答。
- 前端在解析结果后展示 AI 分析工作区。
- B 站字幕优先使用专用 `x/v2/dm/view` 接口获取 CC 字幕 / AI 字幕，覆盖页面存在字幕记录但 `subtitle_url` 为空的公开视频。

一期暂不实现：

- 音频 ASR 转写兜底。无字幕视频返回明确提示：“当前视频没有可提取字幕，音频转写将在后续版本支持。”
- 数据库存储、用户体系、会员权限、任务队列。
- 接入 BibiGPT 或 NoteGPT API；竞品仅作为产品交互和能力参考。

配置边界：

- DeepSeek API Key 通过 `DEEPSEEK_API_KEY` 环境变量提供，不写入仓库。
- AI 分析只面向公开或用户有权访问的视频内容，不改变既有合规边界。
