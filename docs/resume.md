# 简历与面试速览

## 项目一句话

一手遮天视频下载总结器是一个面向公开视频的全栈工具，支持视频链接解析下载、字幕提取、AI 摘要、思维导图、视频问答、账号登录和 Stripe Pro 会员权益。

## 简历写法参考

**一手遮天视频下载总结器 | Vue 3 + FastAPI + yt-dlp + DeepSeek + Stripe**

- 独立完成一个公开视频下载与 AI 总结工具，支持视频解析、格式选择、字幕提取、AI 摘要、思维导图和视频问答。
- 使用 FastAPI 封装 yt-dlp 解析下载能力，并针对 B 站、抖音等平台补充兜底解析、封面代理、视频预览和字幕提取流程。
- 接入 DeepSeek 与 SiliconFlow ASR，实现基于字幕的 AI 总结、长视频内容结构化、流式输出和问答上下文管理。
- 设计账号登录、免费额度、Pro 会员和 Stripe Checkout/Webhook 权益闭环，使用 SQLite 存储用户、会话、订单、权益和 AI 使用次数。
- 完成 SEO/GEO 基础文件、接口测试、项目文档和合规边界说明，提升项目可读性和开源展示质量。

## 可以重点讲的技术点

1. URL 兼容处理：支持标准链接、无协议域名、分享文案中混杂链接的提取。
2. 平台解析策略：通用场景优先使用 yt-dlp，特殊平台通过兜底服务补齐标题、封面、格式、字幕和预览能力。
3. AI 总结链路：优先使用平台字幕；无字幕时走 ASR；再调用大模型生成摘要、大纲、知识点、思维导图和建议问题。
4. SSE 流式体验：AI 总结和问答支持边生成边展示，降低长视频分析等待感。
5. 权益模型：免费用户限制 AI 总结次数，Pro 会员不限次数；后端只信任登录 token，不信任前端传入邮箱。
6. 支付闭环：Stripe Checkout 创建订单，Webhook 校验签名后写入本地权益，并防止已开通用户重复支付。
7. 合规边界：项目只面向用户有权保存的公开视频，不做 DRM、会员墙、私密内容和公共 Cookie 绕过。

## 面试回答思路

**为什么做这个项目？**

公开视频下载只是入口，真正的价值是把长视频变成可复习、可检索、可追问的学习资料，所以我把下载、字幕、AI 总结、导图和问答串成了一个完整工作流。

**项目最难的地方是什么？**

难点不在单个 API，而在真实链路的稳定性：不同平台返回结构不同，视频可能没有字幕，AI 总结耗时较长，支付和权益还要保证后端可信。因此我把解析、字幕、ASR、AI、计费拆成独立服务，并在关键路径补了测试。

**如何保证合规？**

产品和接口都只面向公开或用户有权访问的视频，不提供 DRM 绕过、会员墙绕过、私密内容下载、自动读取 Cookie 或公共账号 Cookie 注入能力，并在 README、AI 可读文件和页面文案中明确说明。
